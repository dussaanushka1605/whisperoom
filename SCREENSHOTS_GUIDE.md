## üì∏ Recommended Screenshots to Add

Here are the screenshots you should add to your README to showcase your Whisperoom application:

### 1. **Login/Signup Page** (High Priority)
**What to capture:**
- The login form with email and password fields
- The "Sign Up" tab showing registration form
- Clean, centered design
- Application title "Whisperoom"

**Why:** First impression - shows the entry point of your application

---

### 2. **Admin Dashboard** (High Priority)
**What to capture:**
- Dashboard showing groups list
- "Create New Group" button/form
- Group cards with names, codes, and member counts
- Navigation to Inbox, Reports, etc.

**Why:** Shows admin capabilities and main interface

---

### 3. **Group Chat Interface** (High Priority)
**What to capture:**
- Active group chat with messages
- Message bubbles (sent/received)
- Input field with emoji, file upload, auto-delete buttons
- Group header showing group name and member count
- Blocked, Report, Theme buttons in header

**Why:** Core feature - shows the main chat functionality

---

### 4. **File Sharing Feature** (Medium Priority)
**What to capture:**
- File upload button/icon
- A message showing a shared file (with file name, size, download button)
- File content preview (if visible)

**Why:** Shows file sharing capability

---

### 5. **Theme Selection** (Medium Priority)
**What to capture:**
- Theme selection dialog/popover
- Multiple color theme options (default, blue, green, purple, etc.)
- Group chat with a non-default theme applied

**Why:** Shows customization feature

---

### 6. **Auto-Delete Messages** (Medium Priority)
**What to capture:**
- Message with "Auto-deletes" or "Deletes in Xs" indicator
- Auto-delete settings popover (clock icon)
- Timer countdown visible on message

**Why:** Shows unique auto-delete feature

---

### 7. **Block/Report Dialogs** (Low Priority)
**What to capture:**
- Block user dialog
- Report user dialog with reason and description fields
- Blocked users list dialog

**Why:** Shows user management and safety features

---

### 8. **Admin Inbox (Chatbot)** (Low Priority)
**What to capture:**
- Admin inbox showing list of users
- Chat conversation with a user
- Message exchange between admin and user

**Why:** Shows admin-user communication feature

## üìù How to Add Screenshots to README

1. Take screenshots using:
   - Windows: `Win + Shift + S` (Snipping Tool)
   - Mac: `Cmd + Shift + 4`
   - Browser: Right-click ‚Üí Inspect ‚Üí Screenshot

2. Save screenshots in a `screenshots/` folder in your project root

3. Add to README.md in the Screenshots section:

```markdown

### Login/Signup Page
![Login Page](./screenshots/login.png)

### Admin Dashboard
![Admin Dashboard](./screenshots/admin-dashboard.png)

### Group Chat Interface
![Group Chat](./screenshots/group-chat.png)

### File Sharing
![File Sharing](./screenshots/file-sharing.png)

![Theme Selection](./screenshots/theme-selection.png)
```

## üí° Tips for Good Screenshots

- **Use consistent browser window size** for all screenshots
- **Hide sensitive data** (emails, real names) if needed
- **Show active features** - make sure features are visible and working
- **Clean interface** - close unnecessary tabs/windows
- **Good lighting** - ensure text is readable
- **Annotate if needed** - add arrows or labels to highlight features
